# 数字员工唯一工作模型

目标：把所有平台工作收敛成一件事——维护一个能解决真实问题的数字员工。

## 你只做的事

1. 在 `one-worker-os` 里维护员工花名册。
2. 为员工写灵魂文档，例如 `<employee>/<employee>.md` 或 `SOUL.md`。
3. 维护员工 README、skills、SOP 和失败复盘。
4. 判断员工是否已经稳定到可以上架收费。

## 平台自动做的事

1. 从 `employee-roster.md` 同步员工目录。
2. 检查员工是否有灵魂文档。
3. 为员工生成版本快照。
4. 管理员审核后上架员工。
5. 用户按月雇佣员工。
6. 支付成功后创建独立实例。
7. 实例生成 Hermes Profile 和微信激活二维码。
8. 后台查看实例、付款、激活和运行状态。

## 每次迭代员工

1. 修改员工灵魂、README 或 skills。
2. 打开 `/admin/bots`。
3. 点击“同步员工”。
4. 检查员工版本是否更新。
5. 确认稳定后设为“已上架”。

## 当前原则

- 平台不生成员工灵魂。
- 平台不改写员工灵魂。
- 新用户雇佣时使用最新员工版本。
- 旧实例是否升级由管理员决定。
- 第一阶段只打磨一个能解决问题的员工，不追求员工数量。
