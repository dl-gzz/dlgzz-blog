# 本地客户端安装说明

## 一句话

- 商店默认安装目标：`http://127.0.0.1:19527`
- 线下客户端白板入口：`http://127.0.0.1:19527/zh/whiteboard`
- 不再混用 `3001`、`3002`、`localhost`

## 商店端默认行为

商店端会优先按下面顺序决定“安装到客户端”要发往哪里：

1. `NEXT_PUBLIC_LOCAL_CLIENT_ORIGIN`
2. `LOCAL_CLIENT_ORIGIN`
3. 默认值 `http://127.0.0.1:19527`

当前代码默认值位置：

- `src/lib/local-client-origin.ts`

## 推荐环境变量

如果线上商店要明确指定安装目标，统一填写：

```bash
NEXT_PUBLIC_LOCAL_CLIENT_ORIGIN=http://127.0.0.1:19527
LOCAL_CLIENT_ORIGIN=http://127.0.0.1:19527
```

## 用户侧要求

用户电脑上需要有线下客户端正在运行，并且本地网关可访问：

```text
http://127.0.0.1:19527/api/health
```

返回成功后，商店点击“安装到客户端”就会把组件安装到线下。

## 验证路径

### 1. 商店端

打开文章或组件详情页，点击：

- `安装到客户端`

### 2. 线下端

打开：

```text
http://127.0.0.1:19527/zh/whiteboard
```

确认白板里出现对应组件。

## 当前统一规则

- 商店安装目标固定理解为：`127.0.0.1:19527`
- 线下客户端固定入口：`127.0.0.1:19527`
- 后续新文档、新按钮、新说明，不再写 `3001`、`3002`、`localhost`
